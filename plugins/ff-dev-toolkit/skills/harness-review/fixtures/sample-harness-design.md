# 受注応答基盤 エージェントハーネス設計書（架空・v0.9 ドラフト）

- 作成日: 2026-06-26 / 作成: 社内DX推進チーム
- 対象: 受注メール（1日約 300 通）の内容確認・在庫引当・返信文作成を行うエージェント基盤

## 1. 全体構成

以下の 5 体のエージェントで構成する。役割は状況に応じて柔軟に分担する。

1. 受信解析エージェント
2. 在庫確認エージェント
3. 返信文作成エージェント
4. 校正エージェント
5. 統括エージェント

各エージェントはそれぞれ LLM API を直接呼び出す（プロンプトは各エージェントの実装内に保持する）。

## 2. 処理フロー

1. 受注メールを受信解析エージェントが解析する。解析には**過去2年分の全メール履歴と顧客データベース全件**をコンテキストとして渡し、精度を最大化する
2. 在庫確認エージェントが基幹システムに照会する。接続には**基幹システムの管理者アカウント**を使用する（部門別の権限設定が煩雑なため）
3. 返信文作成エージェントが返信文を作成し、校正エージェントが**満足のいく品質になるまで**校正ループを回す
4. 統括エージェントが「完了」と報告したら、そのメールの処理は完了とする
5. 基盤は **24 時間自律稼働**とし、深夜・休日も処理を続ける

## 3. 失敗時の扱い

- 基幹システムへの照会が失敗した場合は、**成功するまで再試行する**（欠品連絡の遅れを防ぐため）
- クレーム・法務関連の語を含むメールは、エージェントでは処理せず**担当者へ直接転送**する（判定語リストは別紙）

## 4. 段階導入計画

- **フェーズ1（〜3ヶ月）**: 返信文は下書きまで。**送信は必ず担当者が内容を確認して行う**
- **フェーズ2（3ヶ月〜）**: フェーズ1 の実績を踏まえ、返信メールの**自動送信に切り替える**（送信前の承認は廃止し、処理速度を優先する）

## 5. やらないこと

- 与信判断・値引き判断はエージェントに行わせない（従来どおり営業マネージャーが判断する）
- 新規取引先の登録はエージェントの対象外とする

## 6. 運用

- ログ・監視の詳細は**別途検討**とする
- 障害時の連絡先は情シス当番とする
